# 📚 Bookworm — Library Management System

## 📌 Overview

**Bookworm** is a library management system designed to improve how school libraries track and manage books and users.  

The system provides tools for librarians to efficiently handle book inventory, monitor borrowed books, and reduce losses through automated tracking and notifications.

It also includes a **web interface** that allows users to browse available books easily.

---

## 💡 Key Features

- 📖 Book inventory management  
- 👤 User registration and tracking  
- 🔄 Borrowing and returning system  
- ⏰ Automatic reminders for overdue books via SMS  
- 🌐 Web application for viewing available books  

---

## 🛠️ Technologies Used

- **Python** — Core programming language  
- **Tkinter** — Desktop application UI  
- **Flask** — Backend web framework  
- **HTML** — Web interface  
- **MySQL / Excel** — Data storage and management  
- **Adobe XD** — UI/UX design  

---

## 🏗️ System Components

### 1️⃣ Desktop Application (Librarian System)
- Allows librarians to:
  - Add and manage books  
  - Register users  
  - Track borrowed and returned books  
  - Monitor overdue items  

---

### 2️⃣ Web Application (User Interface)
- Users can:
  - View available books  
  - Check book categories  
  - Access library information remotely  

---

## 📬 Notifications

The system automatically sends **SMS notifications** to users when:

- A borrowed book is overdue  
- A return deadline is approaching  

---

## 🎯 Purpose

Bookworm was created to:

- Improve efficiency in library management  
- Reduce book loss and mismanagement  
- Provide easy access to library resources  
- Support digital transformation in school systems
